# Commercial Contract Amendment

## Overview

Manage contract amendment process including amendment drafting, version control, impact assessment, and approval routing for contract changes.

**Announce at start:** "I'm using the commercial-contract-amendment skill to manage a contract amendment."

## When to Use This Skill

**Trigger Conditions:**
- Contract variation or change required
- Scope of work modification
- Pricing adjustment needed
- Schedule modification
- Terms and conditions change
- Party detail updates

**Prerequisites:**
- Original contract available
- Amendment scope and details defined
- Affected contract clauses identified
- Counterparty agreement obtained

## Step-by-Step Procedure

### Step 1: Amendment Initiation

1. Identify the contract requiring amendment
2. Document the reason for amendment
3. Identify affected contract clauses and sections
4. Determine amendment type (modification, addition, deletion)
5. Assign amendment number and reference

### Step 2: Impact Assessment

1. **Financial Impact:** Calculate cost changes (increase/decrease)
2. **Schedule Impact:** Determine time extensions or compressions
3. **Scope Impact:** Identify scope additions or deletions
4. **Obligation Impact:** Identify new or modified obligations
5. **Risk Impact:** Assess new or changed risk exposures

### Step 3: Draft Amendment Document

1. Use standard amendment template
2. Include contract reference details
3. State the amendment purpose clearly
4. List each modification with original and revised text
5. Include financial schedule adjustments
6. Include effective date
7. Include signature blocks for authorized parties

### Step 4: Version Control

1. Increment contract version number
2. Log amendment in contract register
3. Update affected obligation entries
4. Archive previous version
5. Notify all stakeholders of updated version

### Step 5: Approval Routing

1. Route amendment through approval workflow
2. Track approval status
3. Incorporate approval comments
4. Obtain final signatures

## Success Criteria

- [ ] Amendment reason documented
- [ ] Affected clauses identified
- [ ] Impact assessment completed for all dimensions
- [ ] Amendment document drafted with clear modifications
- [ ] Version control updated
- [ ] Obligation register updated
- [ ] Approval workflow completed
- [ ] Stakeholders notified

## Common Pitfalls

1. **Scope Creep** — Amendments that silently expand scope. Clearly define and limit amendment scope.
2. **Cascading Changes** — One amendment may trigger changes in related clauses. Review all affected areas.
3. **Obligation Gaps** — Amendments may create unassigned obligations. Update obligation register immediately.
4. **Version Confusion** — Multiple amendments create version confusion. Number and track all amendments sequentially.
