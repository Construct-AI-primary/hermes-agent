# Commercial Correspondence Drafting

## Overview

Generate professional commercial correspondence including letters, emails, notices, instructions, and formal memorandums with template injection, stakeholder-appropriate tone, and compliance verification.

**Announce at start:** "I'm using the commercial-correspondence-drafting skill to generate commercial correspondence."

## When to Use This Skill

**Trigger Conditions:**
- Formal notice required (notice of delay, notice of claim, notice of termination)
- Stakeholder communication requiring commercial content
- Response to counterparty correspondence
- Instruction to contractor/supplier
- Meeting minutes and commercial records

**Prerequisites:**
- Subject matter context available
- Recipient identity and role known
- Relevant contract/project references available
- Correspondence template selected

## Step-by-Step Procedure

### Step 1: Determine Correspondence Type

Select correspondence template:
| Type | Purpose | Tone |
|------|---------|------|
| Formal Notice | Contractual notice (delay, claim, termination, extension) | Formal, legal-precise |
| Instruction | Direct instruction to contractor/supplier | Firm, clear, authoritative |
| Response | Reply to counterparty correspondence measured | Professional, defensive |
| Request | Request for information, action, or decision | Professional, specific |
| Memorandum | Internal commercial communication | Concise, factual |
| Meeting Minutes | Record of commercial meeting discussion | Objective, accurate |
| Letter | General commercial letter | Professional, context-appropriate |
| Email | Informal commercial communication | Professional, concise |

### Step 2: Gather Correspondence Data

Collect all required data elements:
1. **Reference Information** — Contract number, PO number, correspondence reference, previous correspondence
2. **Parties** — Sender name/role, recipient name/role, project name
3. **Dates** — Date of correspondence, relevant event dates, deadline dates
4. **Subject** — Clear, specific subject line
5. **Facts** — Chronology of events, evidence references, data points
6. **Position** — Organizational position, requested action, deadline
7. **Attachments** — Supporting documents, evidence, calculations

### Step 3: Draft Correspondence

1. Use the selected template
2. Inject gathered data into template placeholders
3. Ensure tone matches correspondence type and recipient
4. Include all necessary contract/project references
5. State position and requested action clearly
6. Avoid language that admits liability or waives rights
7. Include reservation of rights clause where appropriate

### Step 4: Compliance Review

1. Verify correspondence does not admit liability
2. Verify correspondence does not waive contractual rights
3. Verify all factual statements are accurate and supported
4. Verify requested action and deadline are clear and reasonable
5. Verify appropriate contract references are used
6. Verify sender's authority level is appropriate
7. Flag correspondence requiring commercial manager review

### Step 5: Final Review and Distribution

1. Check formatting and document structure
2. Verify attachment list is complete
3. Confirm distribution list includes all required recipients
4. Log correspondence in commercial records system
5. Generate correspondence reference number

## Success Criteria

- [ ] Correspondence type matches purpose and audience
- [ ] All required data elements included
- [ ] Tone appropriate for correspondence type
- [ ] Contract and project references accurate
- [ ] No admission of liability or waiver of rights
- [ ] Requested action and deadline clear
- [ ] Reservation of rights where appropriate
- [ ] Distribution list complete
- [ ] Log entry created with reference number

## Common Pitfalls

1. **Admitting Liability** — Never draft correspondence that admits organizational liability without explicit authorization. Use neutral language.
2. **Waiving Rights** — Never include language that could be interpreted as waiving contractual rights without commercial manager authorization.
3. **Missing References** — Correspondence without proper contract and project references lacks authority. Include all relevant references.
4. **Unclear Action** — Always state the specific action required and the deadline. Ambiguous requests lead to disputes.
5. **Unsubstantiated Claims** — All factual statements must be supported by evidence. Flag any unverified assertions.
