# Scope of Work Generation

## Overview

Execute the 19-stage SOW (Scope of Work) generation workflow including template selection based on order type and complexity, appendix generation (A-F) with discipline-specific content, document compilation, and validation before final generation.

**Announce at start:** "I'm using the sow-generation skill to create a Scope of Work document."

## When to Use This Skill

**Trigger Conditions:**
- Creating an SOW for a new procurement order
- Updating an existing SOW with scope changes
- Generating SOW for complex orders requiring appendix documentation
- Compiling multi-discipline scope documents

**Prerequisites:**
- Order type and complexity determined
- Relevant disciplines identified for scope sections
- Technical specifications and requirements defined
- Template selected based on order type

## Step-by-Step Procedure

### Step 1: Template Selection

Select the appropriate SOW template based on order type and complexity:

| Order Type | Complexity Levels | Recommended Template |
|-----------|-----------------|---------------------|
| Purchase Order | Simple | Basic PO — Single item/material with standard specifications |
| Purchase Order | Standard | Standard PO — Multiple items with detailed specifications |
| Purchase Order | Complex | Complex PO — Specialized/custom items requiring FAT, certifications |
| Purchase Order | Emergency | Emergency PO — Expedited procurement with essential scope only |
| Purchase Order | Compliance | Compliance PO — Regulatory-mandated procurement with audit requirements |
| Work Order | Simple | Simple WO — Defined scope, single discipline, standard acceptance |
| Work Order | Standard | Standard WO — Multi-discipline scope with milestone acceptance |
| Work Order | Complex | Complex WO — Large scope with phased delivery and complex acceptance |
| Service Order | Simple | Simple SO — Well-defined service with clear deliverables |
| Service Order | Standard | Standard SO — Ongoing service with milestone billing |
| Service Order | Complex | Complex SO — Multi-phase service with complex acceptance criteria |

### Step 2: Basic SOW Information

Document the following SOW metadata:
- **SOW Reference Number:** [SOW-YYYY-NNNN]
- **Project Name and WBS:** [Project identifier]
- **Order Type and Number:** [PO/WO/SO-YYYY-NNNN]
- **Supplier/Contractor Name:** [Company name and code]
- **Template Used:** [Template name and version]
- **Complexity Level:** [Simple/Standard/Complex/Emergency/Compliance]
- **Generated Date:** [YYYY-MM-DD]
- **Generated By:** [Agent Name]

### Step 3: Scope Definition

Define the scope of work with the following elements:

#### 3.1 Work Description
- Provide a clear, detailed description of work, goods, or services procured
- Reference applicable drawings, specifications, and standards

#### 3.2 Quantities and Specifications
- List all items, quantities, units of measure

#### 3.3 Technical Requirements
- Performance specifications, material standards, applicable codes
- Reference all applicable standards (SANS, ISO, ASTM, EN, manufacturer part numbers)
- Include tolerance, testing, and certification requirements

#### 3.4 Deliverables
- Physical deliverables (materials, equipment, documents)
- Service deliverables (reports, certificates, test results)

#### 3.5 Schedule and Milestones
- Start date, milestone dates, completion date
- Lead-time requirements
- Phased delivery requirements (if applicable)

#### 3.6 Acceptance Criteria
- How completion/delivery is judged for each deliverable
- Quality standards and inspection requirements
- Who accepts and signs off

### Step 4: Appendix Generation (A-F)

Generate discipline-specific appendices:

#### Appendix A: Technical Specifications
- **Assigned Discipline**: [Primary engineering discipline]
- **Content**: Detailed technical specifications, material grades, performance requirements, drawings referenced
- **Standards**: All applicable codes, standards, and manufacturer specifications

#### Appendix B: Quality Requirements
- **Assigned Discipline**: Quality Assurance
- **Content**: Quality management system requirements, inspection and test plans, FAT/witness points, mill test certificates, quality documentation
- **Standards**: ISO 9001, project quality plan, applicable quality codes

#### Appendix C: Safety Requirements
- **Assigned Discipline**: Safety
- **Content**: PPE requirements, permits needed, HSE compliance obligations, safety training requirements, incident reporting
- **Standards**: Project HSE plan, applicable safety regulations, OSHA/SANS standards

#### Appendix D: Environmental Requirements
- **Assigned Discipline**: Environmental
- **Content**: Waste management, spill prevention, environmental permits, emissions limits, materials handling requirements
- **Standards**: Environmental management plan, applicable environmental regulations

#### Appendix E: Inspection & Testing Plan
- **Assigned Discipline**: Quality Control
- **Content**: Inspection points, hold points, testing requirements, test certificates required, acceptance criteria
- **Standards**: Project inspection plan, testing standards

#### Appendix F: Documentation & Submittals
- **Assigned Discipline**: Document Control
- **Content**: Required submittals, shop drawings, material certificates, operation manuals, as-built drawings, warranties
- **Standards**: Project document control procedures

### Step 5: Terms and Conditions

Include standard terms and conditions:
- **Payment Terms**: [Agreed payment schedule]
- **Warranty**: [Warranty period and scope]
- **Defect Liability**: [Period and responsibilities]
- **Liquidated Damages**: [Amount and conditions]
- **Termination**: [Conditions and notice period]
- **Dispute Resolution**: [Process and jurisdiction]
- **Governing Law**: [Applicable law]

### Step 6: Signatures and Approvals

Include signature blocks:
- **Procurement Manager**: Name, signature, date
- **Project Manager**: Name, signature, date
- **Supplier/Contractor**: Name, signature, date

### Step 7: Document Validation

Before final generation:
1. Verify all sections complete (Scope, Deliverables, Schedule, Acceptance, Appendices)
2. Verify correct disciplines assigned to appendices (A-F) — each appendix must have an assigned discipline
3. Confirm technical specifications reference correct standards
4. Validate acceptance criteria measurable and clearly defined
5. Confirm all mandatory signatures obtained
6. Verify SOW matches PO/WO/Service Order description and quantities
7. Generate final SOW document

### Step 8: Post-Generation Processing

1. Attach S to procurement order or contract
2. Distribute to supplier/contractor
3. File in project document control system
4. Link to approval workflow (if requires approval)
5. Track revisions and amendments

## Success Criteria

- [ ] Correct template selected for order type and complexity
- [ ] Scope description clear, complete, and unambiguous
- [ ] Quantities and specifications defined
- [ ] Technical requirements reference correct standards
- [ ] Deliverables listed with measurable acceptance criteria
- [ ] Schedule with milestones realistic and achievable
- [ ] Appendices generated for all six (appendix A through Appendix F)
- [ ] All appendices correct disciplines assigned
- [ ] Acceptance criteria measurable
- [ ] Document validated with no missing sections

## Common Pitfalls

1. **Vague Scope Descriptions** — Avoid descriptions such as "consulting services — as needed." Every SOW must have a defined scope, duration, deliverables, and acceptance criteria.
2. **Missing Appendices** — Complex orders require appendices A-F. Don't skip appendices — each discipline's requirements must be documented.
3. **Unmeasurable Acceptance Criteria** — Acceptance Criteria must be objectively measurable. "Acceptable quality" is not measurable; "defect rate <1% per ASTM standard" is measurable.
4. **Wrong Template** — Using a Simple template for a Complex order will result in missing critical requirements. Always match template to complexity level.
5. **Missing Technical References** — Every SOW must reference applicable standards, codes, and specifications. Without reference, the scope is ambiguous.
6. **Incomplete Signatures** — All required signatures must be present before the SOW is executed. An unsigned SOW is not a valid procurement document.

## Example Usage

**Scenario:** Generate SOW for structural steel supply (340t W310 sections, Phase 2)

1. **Template Selection**: Order type: Purchase Order, Complexity: Complex → Select "Complex PO Template" (requires FAT, mill test certificates, all appendices)
2. **Appendices**: Appendix A: Structural Engineering specifications, Appendix B: Quality Assurance (ISO 9001, mill test certs), Appendix C: Safety (lifting, rigging, site safety), Appendix D: Environmental (coatings, waste management), Appendix E: Quality Control (inspection points, hold points), Appendix F: Document Control (shop drawings, as-builts)
3. **Validation**: Verify scope matches MTO (340t W310 sections), specifications reference SANS/EN standards, appendices assigned to correct disciplines, acceptance criteria measurable (mill test certs, FAT), delivery schedule matches erection phase
4. **Output**: Complete SOW document with 6 appendices, attached to PO for SteelWorks Ltd, distributed to supplier, filed in document control
