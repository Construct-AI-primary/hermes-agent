# Innovation Evaluation

## Overview
This skill defines how to evaluate new ideas, technologies, and approaches objectively. Innovation evaluation prevents wasteful investment in unproven concepts while ensuring genuine breakthroughs are identified and pursued.

## When to Use
- When evaluating a new technology, methodology, or approach
- When deciding whether to adopt an emerging trend
- When assessing innovation proposals from teams
- When considering a technology or tool shift
- Before investing significant resources in untested approaches
- **Don't use when:** Making incremental improvements to existing, proven systems

## Core Procedures

### Step 1: Define the Innovation
- **What:** What exactly is new or different?
- **How it works:** Brief explanation of mechanism/approach
- **Origin:** Where did this come from? (research, competitor, startup, etc.)
- **Problem solved:** What specific problem does this address?
- **Current state:** Is this proven, prototype, theoretical, or hype?

### Step 2: Evaluate Adoption Readiness
| Dimension | Question | Score (1-5) |
|-----------|----------|-------------|
| Maturity | Is the technology/approach mature enough? | |
| Evidence | Is there evidence it works? (research, case studies, metrics) | |
| Fit | Does it solve a problem we have? | |
| Cost | Can we afford to adopt it? (implementation, training, migration) | |
| Risk | What happens if it fails? (reversible vs. committed) | |
| Timing | Is now the right time? | |

**Adoption Score = Total / 30**
- 24-30: Strong case for adoption
- 18-23: Worth piloting
- 12-17: Monitor, not ready yet
- Below 12: Likely not viable

### Step 3: Evaluate Impact
- **Potential Benefit:** How much better than current approach?
- **Time to Value:** How long until benefits are realized?
- **Sustainability:** Will this be valuable long-term or is it a fad?
- **Competitive Impact:** Does it give us an advantage or keep us current?

### Step 4: Recommend Action
Based on scores and analysis:
- **ADOPT:** Mature, high-impact, good fit - begin implementation planning
- **PILOT:** Promising but unproven - design small-scale experiment
- **MONITOR:** Early but potentially important - add to technology radar
- **PASS:** Doesn't justify investment - document rationale

### Step 5: Document Decision
```
INNOVATION EVALUATION
=====================
Innovation: [name and brief description]
Adoption Score: [X/30]
Impact Assessment: [benefit, time to value, sustainability]
Recommendation: [adopt/pilot/monitor/pass]
Rationale: [why this decision]
Review Date: [re-evaluate when, if monitor]
```

## Quality Checklist
- [ ] Innovation clearly defined with problem statement
- [ ] Adoption readiness scored across all dimensions
- [ ] Impact assessed for benefit, time, and sustainability
- [ ] Recommendation supported by scores and analysis
- [ ] Decision documented with rationale
- [ ] Review date set for monitored innovations

## Error Handling
- **Error:** Pilot reveals innovation doesn't work as promised
  **Response:** Document findings, share as knowledge, add "didn't work for reason X" to prevent future retries
- **Error:** External factors change evaluation (competitor adopts, technology matures rapidly)
  **Response:** Re-evaluate with current information, update scores and recommendation
